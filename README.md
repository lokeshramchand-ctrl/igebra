# LearnSphere AI

**Personalized AI-Powered Learning Environment**

---

## About The Project

LearnSphere AI transforms static study materials into an interactive, AI-powered knowledge base. By leveraging a Retrieval-Augmented Generation (RAG) architecture, it allows students to query documents, generate personalized assessments, and receive dynamic study recommendations based on their learning history.

## Core Features

* **Verifiable RAG Search:** Ask questions and receive exact answers with document citations and confidence scores to prevent AI hallucinations.
* **Dynamic Assessment Engine:** Automatically generates Multiple Choice, True/False, and Short Answer quizzes directly from uploaded materials.
* **Intelligent Summarization:** Compresses dense, multi-page PDFs into structured "2-Minute Revision Sheets."
* **Proactive AI Tutor:** Analyzes recent chat queries and quiz scores to recommend specific topics and action items for further study.
* **Analytics Dashboard:** Tracks overall engagement, documents ingested, and average quiz performance.

## Tech Stack

* **Frontend:** Next.js (App Router), React, Tailwind CSS
* **Authentication:** NextAuth.js
* **Vector Database:** Supabase (PostgreSQL with `pgvector`)
* **AI Models:** Google Gemini 3.5 Flash
* **Deployment:** Vercel

## Local Development

### 1. Clone & Install

```bash
git clone https://github.com/lokeshramchand-ctrl/igebra.git
cd igebra
npm install

```

### 2. Environment Variables

Create a `.env.local` file in the root directory:

```env
GOOGLE_API_KEY="your_gemini_api_key"
NEXT_PUBLIC_SUPABASE_URL="your_supabase_url"
SUPABASE_SERVICE_ROLE_KEY="your_supabase_role_key"
NEXTAUTH_URL="http://localhost:3000"
NEXTAUTH_SECRET="your_generated_secret_string" 

```

### 3. Database Setup (Supabase)

Run this in your Supabase SQL Editor to configure the vector database and tracking tables:

```sql
create extension if not exists vector;

create table document_chunks (
  id bigint generated by default as identity primary key,
  content text not null,
  metadata jsonb,
  embedding vector(3072)
);

create table quiz_scores (
  id bigint generated by default as identity primary key,
  score int not null,
  total_questions int not null,
  created_at timestamp with time zone default timezone('utc'::text, now())
);

create table chat_logs (
  id bigint generated by default as identity primary key,
  question text not null,
  created_at timestamp with time zone default timezone('utc'::text, now())
);

```

### 4. Run the Application

```bash
npm run dev

```

Open `http://localhost:3000` to view the application.
